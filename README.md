# 🏆 Конкурс Факультетов

Внутренний дашборд для конкурса операторов колл-центра, стилизованный под Хогвартс. Операторы распределены по факультетам (Гриффиндор, Слизерин, Пуффендуй), за каждую из 4 недель им проставляются метрики, по которым считается итоговый балл.

## Возможности

- 📊 Таблица результатов команд и операторов по неделям + итоги месяца
- 🔐 Режим администратора: ручное редактирование метрик и операторов
- 📥 Импорт результатов недели из Excel (.xlsx / .xls / .ods)
- 💾 Хранение в JSON-файле (без БД)
- 🎨 Тёмная Hogwarts-эстетика с SVG-декорациями

## Быстрый старт

Требуется Node.js ≥ 18.

```bash
git clone https://github.com/Dinmukhamad/HarryPotter_Contest.git
cd HarryPotter_Contest
npm install
npm start
```

Откройте http://localhost:3000

Пароль администратора по умолчанию — `hogwarts2026`. **Перед деплоем обязательно смените** через переменную окружения `ADMIN_PASSWORD`.

Все настройки — через переменные окружения (можно положить в `.env`):

| Переменная        | По умолчанию       | Описание                                          |
|-------------------|--------------------|---------------------------------------------------|
| `PORT`            | `3000`             | Порт HTTP-сервера                                 |
| `ADMIN_PASSWORD`  | `hogwarts2026`     | Пароль для записи через `POST /api/state`         |
| `CORS_ORIGIN`     | `*`                | Разрешённый Origin (на проде укажите ваш домен)   |

```
DATA_FILE=/var/data/hogwarts.json
```

## Как считается балл

Балл каждого оператора **берётся напрямую из Excel** (колонка «ИТОГО БАЛЛОВ»). На сайте никаких формул нет — он только показывает то, что пришло из файла.

Балл факультета («Командный зачёт» сверху) — это **среднее** «ИТОГО» по всем операторам факультета, у которых балл не равен 0. Пустые строки (оператор не работал на этой неделе) не размывают среднее.

«Итоги месяца» — среднее недельных средних по тем неделям, где есть данные.

## Формат Excel для импорта

Сайт ожидает Excel в фиксированном формате (как `Таблица_для_конкурса.xlsx`):

- **Строка 1** — большой заголовок (любой текст, пропускается)
- **Строки 2–3** — двухуровневая шапка таблицы
- **Со строки 4** — данные операторов

Фиксированные колонки:

| Excel-колонка | Содержимое                          | Куда попадает на сайте       |
|---------------|-------------------------------------|------------------------------|
| A             | № (порядковый номер)                | игнорируется                 |
| B             | ФИО оператора                       | для поиска оператора         |
| C             | КАЧЕСТВО (значение, например 97.5)  | колонка «Качество»           |
| E             | ОЦЕНКА КЛИЕНТА (например 4.82)      | колонка «Оценка»             |
| G             | % ВЫРАБОТКИ (например 98.85)        | колонка «Выработка %»        |
| I             | ЭФФЕКТИВНОСТЬ % (например 24.12)    | колонка «Эфф. %»             |
| K             | ОПОЗДАНИЯ (минуты)                  | колонка «Опозд. (мин)»       |
| M             | ПОСТОРОННИЕ САЙТЫ (факты)           | колонка «Сайты»              |
| N             | ИТОГО БАЛЛОВ                        | колонка «Итого»              |

Колонки D, F, H, J, L (расчётные «Баллы по сетке» из Excel) на сайт **не загружаются** — они нужны только для вычисления N внутри файла.

### Привязка операторов к факультетам

В Excel нет колонки «Факультет». Распределение задаётся **один раз** через админ-панель:

1. Откройте админ-панель (кнопка «⚯» в углу), войдите.
2. В редакторе добавьте операторов в нужный факультет.
3. При импорте Excel-файла сайт ищет имя из колонки B в списке операторов всех факультетов и кладёт значения в найденного.

**Если имени нет ни в одном факультете — строка пропускается**, в сводке появится список не найденных имён. Добавьте их вручную через админку и повторите импорт.

### Несколько листов в файле

Если в Excel-файле несколько листов, импортируется **только первый** (с предупреждением). Чтобы загрузить вторую неделю — поменяйте в файле порядок листов или переключите выбранную неделю в админ-панели и загрузите другой файл.

### Как импортировать

1. Откройте админ-панель (кнопка «⚯» в правом нижнем углу) и введите пароль.
2. В блоке «📥 Загрузить баллы из Excel» выберите неделю.
3. Прикрепите файл, нажмите «⚡ Применить».
4. После импорта данные сохраняются на сервер автоматически.

## Схема данных

`data.json` имеет три верхнеуровневых поля:

```jsonc
{
  "faculties": [
    {
      "id": "gryf",
      "cls": "gryf",
      "icon": "🦁",
      "crest": "assets/crest-gryffindor.png",
      "name": "Гриффиндор",
      "tagCls": "tag-gryf",
      "scoreCls": "gryf-score",
      "operators": ["Иванов И.И.", "Петров П.П."]
    }
    // … slyth, huff
  ],

  // 4 недели × N факультетов × M операторов × K метрик
  // Последняя метрика — итоговый балл (тип "score")
  "weeklyData": [
    [  // неделя 1
      [  // gryf
        [97.5, 4.82, 98.85, 24.12, 0, 1, 1069.7]
        // …другие операторы
      ],
      [ /* slyth */ ],
      [ /* huff */ ]
    ],
    [ /* неделя 2 */ ],
    [ /* неделя 3 */ ],
    [ /* неделя 4 */ ]
  ],

  "metrics": [
    { "label": "Качество",     "type": "metric"  },
    { "label": "Оценка",       "type": "metric"  },
    { "label": "Выработка %",  "type": "metric"  },
    { "label": "Эфф. %",       "type": "metric"  },
    { "label": "Опозд. (мин)", "type": "penalty" },
    { "label": "Сайты",        "type": "penalty" },
    { "label": "Итого",        "type": "score"   }
  ]
}
```

Типы метрик:
- `metric` — обычный показатель (положительный вклад)
- `penalty` — штраф (отображается со знаком минус)
- `score` — итоговый балл (всегда ровно один, нельзя удалить)

## API

### `GET /api/health`

Проверка живости.

```json
{ "ok": true, "dataFile": "/.../data.json", "time": "2026-05-18T12:00:00.000Z" }
```

### `GET /api/state`

Возвращает текущее состояние. Если файла нет — `{ "state": null }`.

### `POST /api/state`

Перезаписывает состояние целиком. Требует заголовок `X-Admin-Password`.

```bash
curl -X POST http://localhost:3000/api/state \
  -H "Content-Type: application/json" \
  -H "X-Admin-Password: hogwarts2026" \
  -d @data.json
```

Ответ: `{ "ok": true }`, либо `400` при неверной схеме, `403` при неверном пароле.

## Деплой

### Railway / Render / Fly.io

1. Задайте переменные окружения `ADMIN_PASSWORD`, `CORS_ORIGIN`, при необходимости `DATA_FILE` (путь на persistent volume).
2. Команда запуска: `npm start`.
3. Для сохранности `data.json` между деплоями подключите persistent volume.

### Бэкап

`data.json` — единственный источник правды. Настройте регулярное копирование:

```bash
# crontab: каждый час
0 * * * * cp /var/data/hogwarts.json /var/backups/hogwarts-$(date +\%Y\%m\%d-\%H).json
```
